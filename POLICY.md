# 메이플랜드 정책 준수 체크리스트

> 이 프로젝트(버프 타이머)는 게임 정책 **그레이존**에 있음. 개발·기능 추가 시 아래 기준을 매번 확인할 것.

---

## ✅ 반드시 지킬 원칙 (Do)

- [x] **화면 공유(getDisplayMedia) 기반 픽셀 분석만 사용** — 클라이언트/패킷을 절대 건드리지 않음
- [x] **모든 처리는 브라우저 로컬**에서만. 외부 서버로 게임 데이터 전송 금지
- [x] **읽기 전용(read-only)** 유지 — 게임 프로세스·메모리·파일 접근 안 함
- [x] **사용자가 직접 스킬을 시전** — 도구는 "알림"만 제공 (물리적 스톱워치와 동일 위상)
- [ ] 공개 배포 전 **메이플랜드 고객센터 문의**(maple.land) → 공식 입장 확보
- [ ] README/소개 페이지에 **"측정·알림 전용, 자동화 아님"** 명시

## ❌ 절대 금지 (Don't)

- [ ] 키보드/마우스 자동 입력 (매크로성 자동 시전) — **확실한 제재 대상**
- [ ] 게임 클라이언트 파일 읽기/수정
- [ ] 네트워크 패킷 캡처·파싱·수정
- [ ] 게임 메모리 읽기 (DLL 인젝션, 프로세스 접근 등)
- [ ] 오버레이를 게임 창 위에 직접 그리는 기능 (PIP 플로팅 창은 별도 브라우저 창이라 OK)
- [ ] 서버에 게임 데이터(캐릭터명·좌표·전투 로그 등) 업로드
- [ ] 다계정 자동 관리, 자동 사냥·이동 보조 기능 일체

## ⚠️ 추가 기능 검토 시 질문 체크

새 기능 추가 전 아래 질문을 모두 통과해야 함:

1. 이 기능은 게임 클라이언트·메모리·패킷에 **접근하거나 수정하는가?** → YES면 **중단**
2. 이 기능은 **자동으로 게임 입력(키·마우스)을 보내는가?** → YES면 **중단**
3. 이 기능은 **사용자가 하지 못하는 정보**를 제공하는가? (예: 몬스터 HP, 드롭률 등 서버 전용 데이터) → YES면 **중단**
4. 이 기능은 **스톱워치·계산기·메모장으로도 할 수 있는 일**을 자동화한 것인가? → YES면 **OK**
5. 이 기능은 화면에 **이미 표시된 정보**만 읽는가? → YES면 **OK**

## 📜 근거 정책 요약
