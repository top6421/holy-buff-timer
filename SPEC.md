## 1. 프로젝트 개요

### 목적
메이플랜드(MapleStory Worlds 기반)에서 **홀리심볼 버프(2분 지속)**의 잔여시간을 실시간 감지하고, 만료 **5초 전 알림**을 제공하는 웹 서비스.

### 해결하려는 문제
- 버프 지속시간 체크 부담 (항상 시간 확인 어려움)
- 만료 전 갱신해야 효율 유지 (놓치면 경험치 손실)
- 여러 버프 갱신으로 **홀리심볼 아이콘 위치가 수시로 이동**

### 참고 선례
[aprud.me](https://aprud.me) — 동일 원리로 EXP/메소 측정기 운영 중, 제재 사례 0건

## 4. 탐지 알고리즘

### 4.1 3단계 하이브리드 검출

```
[1] 초기 캘리브레이션 (전체 화면 스캔 1회)
      ↓ 아이콘 발견
[2] ROI 자동 설정 (X 크게, Y 타이트)
      ↓
[3] ROI 내 템플릿 매칭 (1초 주기)
      ↓
[4] 회색 비율 + OCR로 잔여시간 추정
      ↓
[5] ≤5초 → 알림
```

### 4.2 템플릿 매칭 파라미터 (검증 완료)

| 항목 | 값 | 근거 |
|---|---|---|
| **템플릿 이미지** | `image/reference/icon.png` (32×32 RGBA) | 실제 게임 에셋, 영상 간 일반화 우수 |
| **고정 스케일** | 3.5× (112×112로 매칭) | 레티나 3316×2046 기준 측정값 |
| **매칭 알고리즘** | `TM_CCOEFF_NORMED` | OpenCV 표준 정규화 상관 |
| **매칭 임계값** | **0.45** | 회색 진행 시 최저 0.547 + 여유 |
| **초기 스캔 스케일 범위** | 1.0 ~ 4.5 (0.25 step) | DPI/해상도 차이 대응 |

### 4.3 ROI 자동 설정 공식

아이콘 발견 위치 `(x, y)`, 크기 `(w, h)` 기준:

```js
ROI = {
  x0: max(0, x - 900),         // 좌측 확장 — 버프 6~7개 추가 수용
  x1: min(W, x + w + 200),     // 우측 약간 여유
  y0: max(0, y - 20),          // Y 타이트 (드리프트만 허용)
  y1: min(H, y + h + 20),
}
```

**근거:** 버프 추가는 X축으로만 일어나고 Y축은 고정 → Y는 타이트하게, X는 크게.

### 4.4 잔여시간 추정 (이중화)

**Primary: 매칭 점수 기반 연속 추정**
```
t=20s(밝음)  → score 0.774
t=60s(절반) → score 0.667
t=100s(80%) → score 0.547
t=115s(만료직전) → score 0.578
```
- 단조 감소 추세, 60초 이후 약간 흔들림 있음
- **보정 필요**: 비선형이므로 OCR로 정밀값 확보

**Secondary: OCR 정밀 측정 (잔여 ≤60초)**
- 60초 이하 구간에서 아이콘에 **큰 노란색 숫자** 표시
- Tesseract.js로 해당 숫자 영역 크롭 후 OCR
- 정확한 초 단위 획득

### 4.5 성능 측정값

| 연산 | 시간 |
|---|---|
| 초기 전체 스캔 (14개 스케일) | ~4200ms |
| ROI 스캔 (고정 스케일) | **~7ms** |
| OCR (Tesseract Worker 재사용) | ~200ms |
| 전체 루프 주기 | 1초 권장 |

## 6. 상태 머신

```
[IDLE]
  ↓ "화면 공유 시작" 클릭
[SHARING]
  ↓ "탐지 시작" 클릭
[SCANNING]  (전체 화면 템플릿매칭, 1초 간격)
  ↓ score ≥ 0.45 & 3회 연속 감지 (오탐 방지)
[TRACKING]  (ROI 내 매칭 + 잔여시간 추정)
  ├─ 매칭 성공 → 잔여시간 업데이트
  ├─ 잔여 ≤ 5초 → [ALERTING]
  ├─ 매칭 실패 10회 연속 → [SCANNING]으로 복귀 (위치 재탐색)
  └─ 아이콘 사라짐 감지 → 버프 종료 처리
[ALERTING]
  ├─ 비프음 재생
  ├─ 브라우저 알림
  ├─ 화면 플래시
  └─ [TRACKING] 복귀 후 갱신 감지 대기
```

### 갱신 감지
- 매칭 점수가 **급격히 상승**(예: 0.55 → 0.77) → 사용자가 갱신 시전
- 잔여시간 카운터 리셋

### 알림 방식 (동시 발동)
1. **Web Audio**: 0.5초 비프 × 3회

3. **화면 플래시**: PIP 창 배경 빨간색 점멸
4. (선택) **진동**: 모바일 `navigator.vibrate()`

### PIP 모드
- Document PIP API로 플로팅 창
- 게임 위에 겹쳐 표시 가능
- 잔여시간 + 색상 경고만 간결하게

## 9. 파라미터 상수 (코드 이관용)

```js
// detector.js 예정
const CONFIG = {
  MATCH_THRESHOLD: 0.45,           // 템플릿매칭 신뢰 임계
  INIT_SCAN_SCALES: [1.0, 1.25, 1.5, 1.75, 2.0, 2.25, 2.5, 2.75,
                     3.0, 3.25, 3.5, 3.75, 4.0, 4.25, 4.5],
  TRACKED_SCALE_STABLE: true,      // 초기 감지 후 스케일 고정
  ROI_PAD_LEFT_PX: 900,            // 버프바 X축 좌측 여유
  ROI_PAD_RIGHT_PX: 200,           // X축 우측 여유
  ROI_PAD_Y_PX: 20,                // Y축 드리프트 허용
  SCAN_INTERVAL_MS: 1000,          // 감지 루프 주기
  CONSECUTIVE_DETECT_REQUIRED: 3,  // 초기 확정에 필요한 연속 성공
  CONSECUTIVE_MISS_RESCAN: 10,     // ROI 실패 시 전체 재스캔 조건
  ALERT_THRESHOLD_SECONDS: 5,      // 만료 몇 초 전 알림
  BUFF_DURATION_SECONDS: 120,      // 홀리심볼 지속시간 (레벨9 기준)
  TEMPLATE_PATH: "image/reference/icon.png",
};
```

## 11. 미해결 / 향후 검증 필요
